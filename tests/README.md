# Tests Overview

This folder holds test planning and implementation for the MCP-first extension.

## Structure

- `tests/unit/` for unit tests and unit test plans.
- `tests/integration/` for integration tests and plans.
- `tests/e2e/` for end-to-end tests and plans.

## Test IDs

- Unit tests use `UT-###`.
- Integration tests use `IT-###`.
- End-to-end tests use `E2E-###`.

## Ownership

- QA and release coordination owns test plans and scaffolds.
- Runtime and UI owners implement test code in their respective surfaces.
